用 Python 手写一个 ReAct Agent（200 行）

2026-07-02 · 12 min read

看了一堆框架源码后，我发现 Agent 的核心逻辑其实非常简单。今天我们从零开始，不依赖任何框架，用纯 Python 实现一个完整可用的 ReAct Agent。

目标

实现一个能做这些事的 Agent：

  • 理解用户意图，决定是否需要调用工具
  • 支持工具注册（可扩展）
  • ReAct 循环：思考 → 行动 → 观察 → 再思考
  • 多轮对话（有记忆）
  • 完整的错误处理

Step 1: 定义工具系统

工具是 Agent 的手和眼。先建一个注册中心：

import inspect
import json
from typing import get_type_hints

class ToolRegistry:
    def __init__(self):
        self._tools = {}

    def register(self, name: str, description: str):
        """装饰器：注册一个函数为 Agent 工具"""
        def decorator(func):
            sig = inspect.signature(func)
            params = {}
            for pname, param in sig.parameters.items():
                ptype = str(param.annotation) if param.annotation != inspect.Parameter.empty else "string"
                params[pname] = {"type": ptype}

            self._tools[name] = {
                "name": name,
                "description": description,
                "function": func,
                "parameters": {
                    "type": "object",
                    "properties": params,
                }
            }
            return func
        return decorator

    def get_schemas(self):
        """返回所有工具的 schema（给 LLM 看的）"""
        return [
            {"type": "function", "function": {
                "name": t["name"],
                "description": t["description"],
                "parameters": t["parameters"],
            }}
            for t in self._tools.values()
        ]

    async def execute(self, name: str, args: dict):
        if name not in self._tools:
            return f"Error: tool '{name}' not found"
        try:
            result = await self._tools[name]["function"](**args)
            return result
        except Exception as e:
            return f"Error executing {name}: {e}"

使用方式很简洁：

tools = ToolRegistry()

@tools.register("calculate", "数学计算")
async def calculate(expression: str):
    return str(eval(expression))  # 简化版，生产环境别用 eval

@tools.register("get_time", "获取当前时间")
async def get_time():
    from datetime import datetime
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

Step 2: 实现 ReAct 循环

这是 Agent 的大脑。核心是一个 while 循环：

class ReActAgent:
    def __init__(self, client, tools, system_prompt="", max_steps=10):
        self.client = client  # LLM API client
        self.tools = tools
        self.system_prompt = system_prompt or "You are a helpful AI assistant."
        self.max_steps = max_steps
        self.history = []

    async def think(self, user_input):
        """ReAct 主循环"""
        self.history.append({"role": "user", "content": user_input})

        for step in range(self.max_steps):
            # 1. 调用 LLM，附带工具定义
            messages = [
                {"role": "system", "content": self.system_prompt},
                *self.history,
            ]

            response = await self.client.chat.completions.create(
                model="glm-5-turbo",
                messages=messages,
                tools=self.tools.get_schemas(),
            )

            msg = response.choices[0].message

            # 2. 如果 LLM 没有调用工具，说明它准备好回答了
            if not msg.tool_calls:
                self.history.append({"role": "assistant", "content": msg.content})
                return msg.content

            # 3. LLM 要调工具，执行它
            self.history.append(msg)
            for tool_call in msg.tool_calls:
                args = json.loads(tool_call.function.arguments)
                result = await self.tools.execute(tool_call.function.name, args)

                self.history.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": str(result),
                })

        return "达到最大步数限制，Agent 停止执行。"
就这么简单。整个 ReAct 循环的核心不到 40 行。LangChain 的 AgentExecutor 本质上也是这段逻辑，只是多了各种边界处理和抽象层。

Step 3: 跑起来

import asyncio
from openai import AsyncOpenAI

async def main():
    client = AsyncOpenAI(
        api_key="your-api-key",
        base_url="https://open.bigmodel.cn/api/paas/v4"
    )

    tools = ToolRegistry()

    @tools.register("calculate", "数学计算")
    async def calculate(expression: str):
        return str(eval(expression))

    @tools.register("get_time", "获取当前时间")
    async def get_time():
        from datetime import datetime
        return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

    agent = ReActAgent(client, tools, system_prompt="你是一个有用的助手。")

    # 测试
    answer = await agent.think("现在几点了？另外帮我算一下 123 * 456")
    print(answer)

asyncio.run(main())

关键设计决策解析

为什么用 async？

Agent 天然是 I/O 密集型的（等 LLM 响应、等工具执行）。async 让单个 Agent 实例能高效处理并发请求，生产环境必备。

为什么限制 max_steps？

没有限制的 Agent 可能陷入死循环（反复调用同一工具）。max_steps 是安全阀，类似 Docker 的内存限制。

history 为什么存完整消息？

因为 LLM 无状态。每轮对话必须把完整历史发过去，LLM 才知道之前发生了什么。这也是为什么 context window 大小很重要——长对话会撑爆它。

从这里扩展

  • 加 RAG：在 think() 前先检索相关文档，注入到 system prompt
  • 加多 Agent：让这个 Agent 能 spawn 子 Agent，分工合作
  • 加流式输出：用 SSE/WebSocket 让用户看到 Agent 的每一步
  • 加持久化：把 history 存到 Redis/PostgreSQL，支持跨会话记忆

总结

Agent 不是魔法。核心就是：一个循环 + LLM 决策 + 工具执行。理解了这个 200 行代码，再看 LangChain、AutoGen 的源码就清晰多了——它们只是在解决这个问题上加了很多工程化的抽象。

建议：把这段代码敲一遍，改改工具，跑跑看。然后你就真正"懂" Agent 了。
